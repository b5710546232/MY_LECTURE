Association rules discovery

เป็นสิ่งที่ machine learning ไม่มี

  • 1993, Agrawal (of IBM)
  • การเพิ่มยอดขายของ
  • เริ่มจากการเก็บข้อมูลว่าลูกค้าซื้ออะไร
  • สนใจข้อมูล categorical แม้ว่าข้อมูลจะเป็นตัวเลขก็ตาม
  • รายการที่ลูกค้าซื้อ=transaction พี่วิน = {น้ำดื่ม ,ผ้าอ้อม}

พี่ภูมิ={เบียร์, ผ้าอ้อม} พี่ผึ้ง={เบียร์}

หาโอกาสที่ลูกค้าจะซื้ออะไรกับอะไร ของที่คนซื้อไว้ด้วยกัน แล้วแต่เรา(คน)จะต้องตัดสินใจเองว่าจะไว้ของใกล้กันหรือห่างกัน ระบบนี้ทำหน้าที่ช่วยตัดสินใจให้ แต่ไม่ได้ตัดสินใจให้

  • ทุกกฎที่ได้มาจากการสำรวจก็ใช่ว่าจะดี มันจะมีค่า support(บ่งบอกถึงความถี่ มีทั้งต้นและปลาย,จำนวนtransactionที่มีของ/จำนวน transactionทั้งหมด) กับค่า confident (ค่าความเชื่อมั่น,จำนวนการทำตามกฎ) ยิ่งมากยิ่งดี

กำหนดให้

  • I=sample space={i1,i2,i3,….,im }
  • T=transaction={t1,t2,t3,…,tn}, ti = subset ของ I
  • rule x(antecedence)=>y(consequence) โดยที่ x,y เป็นจะต้องเป็นส่วนหนึ่งของ I และ intersect กันได้เซตว่าง (ไม่อินเตอร์เซคกัน)
  • itemset เซตของไอเทม {i1,i2,i5}:3-itemset {i2}:1-itemset
  • k-itemset ไอเทมเซตขนาด k

  • ค่า support คำนวณให้itemset หรือ rule ก็ๆได้ เช่น sup({เบียร์})=⅔ sup({เบียร์} -> {ผ้าอ้อม}) =1/3

  • ค่า confidence คำนวณให้กฎได้อย่างเดียว
  • แบ่งเป็นสองขั้นตอน

ตัวอย่าง ให้ f1={A,B,C} f2={A,C} f3={A,D} f4={B} ให้ min-confidence=0.5, min-support=0.5

  • เราจะได้ I=sample space = {A,B,C,D}
  • เราจะหา Frequent itemset discovery ได้จากการหาค่า support ซึ่งไอเทมที่เราจะหานั้นจะได้จากการที่เราวาด lacttice ออกมา
  • ดู frequence ของแต่ละเซต

  • ความเร็วในการประมวลผลขึ้นกับจำนวนข้อมูลที่เราเอาไปคำนวณ -> lattice ใหญ่=ช้า

  • เราจะทำการตัดช้อยส์ โดยถ้าไอเทมเล็กที่ frequence สูงๆ ไอเทมใหญ่ frequence ก็จะสูงตาม = pronning ตัดกิ่ง มีสองแบบ คือมองจากข้างบน Apriori algorithm และมองจากข้างล่าง(มองจากตัวใหญ่ก่อน) Closed Algorithm

  • จากกฎ {A}: {A}->{} ผิดเพราะซื้อเซตว่างมันคงแปลกๆ, {}->{A} ผิดเพราะซื้อเซตว่างมันคงแปลกๆ เราก็เลยจะไม่สนใจ 1-itemset จะสนใจ 2-itemset ขึ้นไป

    {A,C}: {A}->{C} conf = 2/3 (ยึดคนซื้อ A แล้วซื้อ C)

    {C}->{A} conf = 1 (ยึดคนซื้อ C แล้วซื้อ A)

    ทั้งสองอันนี้เกิน min_support ที่เรากำหนดไว้ 0.5 สามารถกลายเป็นกฎได้

    {A,C}->{}

    { }->{A,C}

ทั้งสองอันนี้ผิดเพราะเซตว่างมันไม่ make sense

  • ถ้าไอเทมสามอันเราจะได้ 6 กฎ {A,B,C}: {A,B}->{C},{A,C}->{B} , {B,C}->{A},{A}->{B,C}, {B}->{A,C} , {C}->{B,A} Lattice สำหรับไอเทมสามอัน
